# Weather Bot（Discord × Playwright）

Discord上で地域名を入力すると、対応する地域の天気情報をスクリーンショット付きで返すBotです。  
Python、discord.py、Playwrightを用いて開発しました。

## 概要

Discordで `!天気 地域名` と入力すると、対象地域の天気ページを取得し、天気表示部分のスクリーンショットを取得してチャンネルに送信します。

### 例
- `!天気 東京`
- `!天気 福岡`

## 使用技術

- Python
- discord.py
- Playwright
- python-dotenv

## 主な機能

- 指定した地域の天気情報を取得し、画像として送信
- 地域未指定時のエラーメッセージ表示
- 未対応地域入力時のガイド表示
- 要素取得失敗時のフォールバック（全画面スクリーンショット）

## 処理の流れ

1. Discordコマンドで地域名を受け取る
2. 「県」「市」などの表記を除去し、入力の揺れを吸収
3. 地域名に対応するURLを辞書（`city_urls.py`）から取得
4. Playwrightで対象ページを開く
5. 天気表示要素（`section.section-wrap`）の読み込みを待機
6. 該当要素をスクリーンショットとして保存
7. Discordチャンネルに画像を送信

## 工夫した点

### 1. 入力の揺れへの対応
- 「県」「市」や全角スペースを除去し、ユーザー入力のばらつきを吸収しています。

### 2. UI依存処理の安定化
- `wait_for_selector` を使用し、要素の描画完了を待ってから取得しています。
- 固定時間待機も併用し、描画遅延への対策を行っています。

### 3. フォールバック設計
- 指定要素が取得できない場合でも処理を止めず、ページ全体のスクリーンショットを送信する仕組みを実装しています。

### 4. 拡張しやすい構成
- 地域とURLの対応を辞書で管理し、対応地域を追加しやすい構成にしています。

### 5. セキュリティへの配慮
- Discordトークンをコードに直接記載せず、環境変数（`.env`）で管理しています。

## 補足：スクリーンショット取得の検証コード

Playwrightの動作確認用として、単体でスクリーンショットを取得するスクリプトも作成しています。

本体は非同期処理（async）で動作しますが、検証用コードでは同期処理（sync）を使用することで、ブラウザ操作やスクリーンショット取得の挙動を切り分けて確認できるようにしています。

## 実行方法

### 1. ライブラリのインストール

```bash
pip install discord.py playwright python-dotenv
```

### 2. 環境変数の設定

-.env ファイルを作成し、以下を記述します。
```bash
DISCORD_TOKEN=your_token_here
```

### 3. Botの起動

```bash
python weather_discord_bot.py
```
