# Weather Bot（Discord × Playwright）

## 概要
Discord上で地域名を入力すると、対応する地域の天気情報をスクリーンショット付きで返すBotです。  
Python、discord.py、Playwrightを用いて開発しました。

本Botは、Webページ上の情報を取得して外部サービスへ返す自動化の実践として作成したものであり、入力処理、UI取得、待機、例外時フォールバックまで含めて設計しています。

## 背景・目的
業務や個人開発で自動化に取り組む中で、処理そのものだけでなく、入力揺れや読み込みタイミング、UI取得失敗時の扱いを含めて考えることが重要だと感じてきました。

そこで本Botでは、Discordから地域名を受け取り、対応する天気ページへアクセスし、必要な表示領域をスクリーンショットとして返す仕組みを実装しました。  
単なる情報取得ではなく、Web UIを対象とした自動化において、取得の安定性や失敗時の振る舞いを意識して設計した点を重視しています。

## 主な機能
- Discordコマンド `!天気 地域名` による実行
- 指定地域の天気情報を取得し、画像として送信
- 地域未指定時のエラーメッセージ表示
- 未対応地域入力時のガイド表示
- 要素取得失敗時のフォールバック（全画面スクリーンショット）
- 環境変数（`.env`）によるDiscordトークン管理

## 処理の流れ
1. Discordコマンドで地域名を受け取る
2. 入力からスペースや「県」「市」などの表記を除去し、入力揺れを吸収する
3. 地域名に対応するURLを `city_urls.py` から取得する
4. Playwrightで対象ページを開く
5. ページロード完了後、対象要素の描画を待機する
6. 天気表示領域をスクリーンショットとして保存する
7. 要素取得に失敗した場合は、フォールバックとしてページ全体のスクリーンショットを取得する
8. Discordチャンネルへ画像を送信する

## 設計上の工夫
- 入力時の「県」「市」や全角スペースを除去し、ユーザー入力のばらつきを吸収
- `wait_for_load_state`、`wait_for_selector`、固定待機を組み合わせ、描画タイミングを考慮して安定化
- 取得対象要素が見つからない場合でも処理を止めず、全画面スクリーンショットを返すフォールバックを実装
- 地域名とURLの対応表を別ファイルで管理し、対応地域を追加しやすい構成に分離
- Discordトークンをコードに直接記載せず、環境変数で管理

## 補足：検証用スクリプト
Playwrightの動作確認用として、単体でスクリーンショットを取得する `weather_screenshot.py` も作成しています。  
本体はDiscord連携と非同期処理を含みますが、検証用スクリプトを分けることで、ブラウザ操作やスクリーンショット取得の挙動を切り分けて確認しやすくしています。

## この開発で認識した課題
本Botは比較的安定したWebページを対象とした自動化ですが、それでも以下のような課題があると認識しています。

- HTML構造やクラス名の変更により、取得対象要素が見つからなくなる可能性がある
- ページ描画タイミングにばらつきがあるため、固定待機に頼ると過不足が生じることがある
- 対応地域は辞書で手動管理しているため、対象拡張にはメンテナンスが必要
- スクリーンショットファイルの整理や削除処理は未実装であり、長期運用を考えると改善余地がある

この経験から、Web自動化においても「対象が比較的安定して見える場合であっても、UI構造や描画タイミングへの依存を避けることは難しい」と実感しました。

## 実行方法

### 1. ライブラリのインストール

```bash
pip install discord.py playwright python-dotenv
```

### 2. 環境変数の設定

-.env ファイルを作成し、以下を記述します。
```bash
DISCORD_TOKEN=your_token_here
```

### 3. Botの起動

```bash
python weather_discord_bot.py
```
